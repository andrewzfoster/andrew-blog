JOIN(INNER JOIN)

  • 只保留两张表中都匹配的记录。

LEFT JOIN

  • 以左表为主，保留左表所有记录，即使右表中没有匹配的项。

RIGHT JOIN

  • 以右表为主，保留右表所有记录，即使左表中没有匹配的项。

IN

  • 是用于 过滤（WHERE 条件） 的一种语法，用来判断某个值是否属于一个给定的列表。

ON

  • 用于 JOIN 连接中，用来指定 两个表之间关联的条件。

UNION ALL

  • UNION 会去重，UNION ALL 不会，性能更高，适合递归查询。

  • 将两个或多个查询的结果拼接在一起（不去重）。

  • 例子：把 employees 和 managers 中的所有 name 合并（包括重复项）。

  • id name
    1 Bob
    2 David
    id name
    1 Bob
    2 David

  • 结果

  • name
    Alice
    Bob
    Charlie
    Bob ← 重复出现
    David

GROUP BY

  • 将结果按照某个字段“分组”，通常配合聚合函数使用（如 COUNT()、SUM() 等）。

  • 例子

  • id user_id org_id
    a1 u1 o1
    a2 u1 o2
    a3 u2 o1
    a4 u3 o3
    a5 u1 o3

  • 结果

  • user_id org_count
    u1 3
    u2 1
    u3 1

AS

  • 不区分大小写(as AS 均可) 可省略
  • 给某个字段或表起一个别名。

WITH RECURSIVE

  • 用于 递归查询 的关键语法，主要用于处理 树状结构 或 图状结构 数据, 可以从某个节点往下找子节点，或从某个节点往上找父节点，直到找完为止。

  • 例子

  • id name parent_id
    1 总公司 NULL
    2 技术部 1
    3 前端组 2
    4 后端组 2
    5 产品部 1
    6 设计组 5
    7 交互组 5

  • 结果:从 技术部（id=2） 向下，查出其所有子部门（包括子孙）

  • id name parent_id
    2 技术部 1
    3 前端组 2
    4 后端组 2
  • 结果:从 前端组（id=3） 向上，查出其所有上级（直到根）
  • id name parent_id
    3 前端组 2
    2 技术部 1
    1 总公司 NULL

OVER

  • 用于窗口函数中，表示“这个函数作用在哪一块数据上”。

PARTITION BY

  • 是 OVER() 的一部分，用来划分窗口的“分区”，就像 GROUP BY 一样，但不聚合。
