获取 ssh 密钥 key 并配置

1.检查自己是否已经生成密钥
密钥文件夹位置：C:\Users\Administrator\.ssh
如果存在 id_rsa(私有密钥)和 id_rsa.pub(公有密钥)则说明已经生成过,可以通过文本直接打开公有密钥并复制或选择生成新的密钥
2.生成密钥
检查自己是否已设置过 git 用户名及邮箱,如果无查询结果,需要先配置用户名及邮箱

生成密钥指令

生成密钥的窗口

到此C:\Users\Administrator\.ssh文件目录下已生成 id_rsa(私有密钥)和 id_rsa.pub(公有密钥),复制公有密钥所有内容

3.项目管理人员将生成的公有密钥添加到项目拥有者的 SSH keys 中

  • 进入 github 下的 Settings 中

  • 进入到 SSH and GPG keys 下并选择 New SSH key

  • Title 用于编辑当前备注,Key Type 选择默认,将第二步复制的所有内容粘贴到 Key 中,点击 Add SSH key 即可

  • 测试是否成功
    通过指令ssh -T git@github.com出现以下结果即为成功,如出现提示输入 yes/no 选择 yes 即可

https 和 ssh 的区别

  • HTTPS 协议只认账号。如果使用 HTTPS 操作远程仓库，则需要使用账号密码来做权限的认证。
    SSH 协议只认机器。当使用 SSH 操作远程仓库的话，需要使用公钥和私钥对来做权限的认证。
  • HTTPS 协议在未保存凭据的时候 push 是需要验证用户名和密码的
    SSH 协议在 push 的时候，除非如果配置 SSH key 的时候设置了密码，否则是不需要输入用户名的

git 配置的用户名和邮箱有什么用？本地用户名和邮箱可以与线上 git 地址配置不一样的吗？

  • 远程仓库为了跟踪是谁提交的代码，需要提交这提供身份信息：邮箱和用户名。
  • 可以,本地 Git 的用户名和邮箱地址主要用于标识提交记录中的作者信息，而远程 Git 服务上的用户名通常用于身份验证和访问控制。此外
    • 如果 Git 配置的邮箱是 GitHub 中存在的邮箱，则 commits 里显示的是这个邮箱对应的账号
    • 如果 Git 配置的邮箱是一个在 GitHub 里不存在的邮箱，则 commits 里显示的是 Git 配置的用户名
